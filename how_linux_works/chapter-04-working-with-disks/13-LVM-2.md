# My question

```
Nice, so why don't we just stick with /dev/dm-0, why /dev/ubuntu-vg/root or /dev/mapper/ubuntu--vg-root ?
```
Good question — this gets to the core of why Linux has _multiple naming layers_ instead of just using `/dev/dm-0`.

Short answer:  
👉 `/dev/dm-0` is **too low-level and unstable**, while `/dev/mapper/...` and `/dev/<vg>/<lv>` are **stable, meaningful, and persistent**.

---

# 🧠 Why not just use `/dev/dm-0`?

## 1. It is NOT stable across reboots

`/dev/dm-0`, `/dev/dm-1`, etc. are assigned dynamically.

Example:

```
boot 1:
  root → /dev/dm-0
  swap → /dev/dm-1
```

After reboot:

```
boot 2:
  root → /dev/dm-1
  swap → /dev/dm-0
```

So the numbering can change depending on:

- boot order
- device detection timing
- udev events
- kernel decisions

👉 That makes `/dev/dm-0` unreliable for configuration.

---

## 2. It has no semantic meaning

This is the biggest issue.

```
/dev/dm-0  ❌ meaningless
```

You don’t know:

- is it root?
- swap?
- database volume?
- backup volume?

It’s just “the first device-mapper device”.

---

## 3. You need human-readable identifiers

LVM adds naming so you can understand what you're mounting:

```
/dev/ubuntu-vg/root
```

Now you instantly know:

- VG = ubuntu-vg
- LV = root

Much safer for:

- `/etc/fstab`
- bootloader configs
- systemd mounts

---

# 🧩 Why `/dev/mapper/...` exists

This is the **official device-mapper naming layer**.

Example:

```
/dev/mapper/ubuntu--vg-root
```

## Why this exists:

### 1. Stable name

Unlike `/dev/dm-0`, this is persistent across boots.

---

### 2. Encodes structure

It tells you:

```
VG = ubuntu-vg
LV = root
```

But encoded into a flat device name.

---

### 3. Works for all DM users

Not just LVM:

- encrypted disks (LUKS)
- RAID
- thin provisioning
- snapshots

All use `/dev/mapper/*`

So it is a **generic interface layer**, not LVM-specific.

---

# 🧩 Why `/dev/ubuntu-vg/root` exists

This is a **convenience symlink created by LVM tools (udev rules)**.

```
/dev/ubuntu-vg/root → /dev/dm-0
```

or sometimes:

```
/dev/ubuntu-vg/root → /dev/mapper/ubuntu--vg-root
```

## Why it exists:

✔ Easier to read  
✔ Matches LVM structure directly  
✔ Nice for admin tools

---

# 🧠 So why 3 names for the same thing?

Because each layer solves a different problem:

|Name|Purpose|Why it exists|
|---|---|---|
|`/dev/dm-0`|kernel internal ID|minimal, fast, but unstable|
|`/dev/mapper/ubuntu--vg-root`|canonical DM interface|stable, system-wide standard|
|`/dev/ubuntu-vg/root`|LVM convenience path|human-friendly alias|

---

# 🔥 Mental model (important)

Think of it like this:

```
dm-0 = "internal pointer"
mapper/ = "official API name"
vg/lv path = "user-friendly shortcut"
```

Or even simpler:

```
dm-0 → machine ID (don’t rely on it)
mapper → system standard name (reliable)
vg/lv → readable alias (for humans)
```

---

# ⚙️ What breaks if you use `/dev/dm-0`?

If you put this in `/etc/fstab`:

```
/dev/dm-0  /  ext4  defaults  0 1   ❌
```

You risk:

- mounting the wrong filesystem
- boot failures
- swapped root/swap devices
- unpredictable behavior

---

# 🧠 Real-world rule

### Always use one of these:

✔ Best practice:

```
/dev/mapper/ubuntu--vg-root
```

✔ Also fine:

```
/dev/ubuntu-vg/root
```

❌ Avoid:

```
/dev/dm-0
```

---

# 🧩 Final intuition

- `/dev/dm-0` = raw engine output
- `/dev/mapper/...` = official interface
- `/dev/vg/lv` = human-readable alias
