# 5.5 GRUB (Grand Unified Boot Loader) - Notes
One of GRUB's most important features is its ability to:

- Navigate filesystems
- Locate kernel images
- Locate initramfs images
- Load GRUB modules dynamically
- Support multiple operating systems

- Hold `SHIFT` (BIOS) or `ESC` (UEFI) to access GRUB menu.
- In Grub menu `e` to access configuration editor.
- `c` To open GRUB shell.
- NOTE: Its a different shell from linux, even though command names are similar, its a whole different world and its just that naming pattern are same.
---
# Example GRUB Boot Entry

A GRUB boot entry contains:
```
set root='hd0,msdos1'

search --fs-uuid ...

linux /boot/vmlinuz-4.15.0-45-generic \
      root=UUID=... \
      ro quiet splash

initrd /boot/initrd.img-4.15.0-45-generic
```
- There are actually **two different roots**.
- `Linux Root Filesystem` passe to the kernel:
```
linux /boot/vmlinuz \
root=UUID=...
```
- which becomes `/` after linux boots.
- `GRUB Root` only inside GRUB.
```
set root=(hd0,msdos1)
```
- This tells grub to look for kernel files here.
- GRUB root and Linux root are completely different concepts.

# GRUB Modules

GRUB is modular.

Modules are loaded using:

```
insmod
```

Example:

```
insmod ext2
insmod part_msdos
insmod gzio
```

Important:

```
GRUB insmod
≠
Linux kernel insmod
```

They are unrelated commands.

---

## LVM Support

Systems booting from LVM require:

```
lvm.mod
```

Example:

```
insmod lvm
```

Without it:

```
GRUB cannot locate kernels
stored on logical volumes.
```

### GRUB device naming
- Not `/dev/sda`
- Instead it used `(hd0)` , `(hd`)`.
- With partitions info:
```
(hd0,msdos1)  
(hd0,msdos2)
```
- GPT Partitions:
```
(hd0,gpt1)  
(hd0,gpt2)
```

---

## Example Menu Entry

```
menuentry 'Ubuntu' {

    insmod ext2

    set root='hd0,msdos1'

    search --fs-uuid --set=root UUID

    linux /boot/vmlinuz-4.15.0-45-generic \
          root=UUID=... \
          ro quiet splash

    initrd /boot/initrd.img-4.15.0-45-generic
}
```

# GRUB Installation
- After building GRUB for MBR or UEFI.

Install GRUB to current disk:

```
sudo grub-install /dev/sda
```

Installs:

- Bootloader code
- Required modules
- Configuration support

---

⚠️ Incorrect use can make the system unbootable.

---

# Installing GRUB on Another Disk

Example:

```
sudo grub-install \
--boot-directory=/mnt/boot \
/dev/sdc
```

Meaning:

```
Install bootloader on /dev/sdc
Use /mnt/boot as target boot directory
```

```

   ↓

   ↓
```

# Shim

Many Linux distributions use:

```
shimx64.efi
```

between UEFI and GRUB.

Flow:

```
UEFI
   ↓
shimx64.efi
   ↓
grubx64.efi
   ↓
Linux Kernel
```

# Chainloading

GRUB can load another bootloader.

This is called:

```
Chainloading
```

---

## Windows Example

```
menuentry "Windows" {

    insmod chain
    insmod ntfs

    set root=(hd0,3)

    chainloader +1
}
```

---

## How chainloader +1 Works

```
Load first sector
of target partition
and execute it
```

Usually loads:

- Windows bootloader
- DOS bootloader
- Another GRUB installation

# MBR Boot Process

Traditional BIOS systems:

```
Power On
   ↓
BIOS
   ↓
MBR (441 bytes)
   ↓
GRUB Stage 1
   ↓
GRUB Stage 2
   ↓
Kernel
```

---

## MBR Limitation

MBR contains only:

```
441 bytes
```

of executable code.

Too small for GRUB.

Therefore:

```
Stage 1
    ↓
Loads Stage 2
```

stored elsewhere on disk.

---

# BIOS Boot Partition

Required when:

```
GPT + BIOS
```

are used together.

Special partition UUID:

```
21686148-6449-6E6F-744E-656564454649
```

Stores GRUB boot code.

---

# UEFI Boot Process

Much simpler.

```
Power On
    ↓
UEFI Firmware
    ↓
EFI System Partition (ESP)
    ↓
grubx64.efi
    ↓
Linux Kernel
```

---

# EFI System Partition (ESP)

Special FAT filesystem.

Usually mounted at:

```
/boot/efi
```

Contains:

```
EFI/
```

directory.

---

## Example Layout

```
EFI/
├── Microsoft/
├── Ubuntu/
├── GRUB/
└── Apple/
```

Example files:

```
grubx64.efishimx64.efi
```

---

# MBR vs UEFI

|Feature|MBR|UEFI|
|---|---|---|
|Boot code location|MBR sectors|EFI files|
|Filesystem aware|No|Yes|
|GPT support|Limited|Native|
|Secure Boot|No|Yes|
|Configuration|Complex|Simpler|
|Modern standard|No|Yes|

# Complete Boot Flow (UEFI + GRUB)

```
Power On
    ↓
UEFI Firmware
    ↓
EFI System Partition
    ↓
shimx64.efi
    ↓
grubx64.efi
    ↓
grub.cfg
    ↓
Load Modules
    ↓
Load Kernel (vmlinuz)
    ↓
Load initramfs
    ↓
Pass Kernel Parameters
    ↓
Linux Kernel
    ↓
systemd/init
    ↓
User Space
```
